Analysis

Responsibilities

Analysis is challenging. Setting up a proper analysis requires a thorough understanding of the system design, it's expected functionality, and boundary conditions. The analysis team at RPL has the obvious goal of analyzing design and providing feedback to the designers, however, getting there is very time consuming.

The Process

Hand Calculations

The first step in analysis at RPL is performing hand calculations. The analysis engineer will setup a simple 1D or 2D representation of the problem and attempt to solve. These solutions are commonly accurate enough to drive design decisions, however not accurate enough to use as models for flight performance or engine performance. The numbers obtained through hand calculations are generally signifiantly easier to use as a logic check than those obtained through computational methods.

Computational Methods

When more accurate analysis is needed, RPl performs analysis using computational methods such as Computational Fluid Dynamics (CFD) and Finite Element Analysis (FEA). The code necessary to solve these complex multidimensional problems is availabe in softare such as ANSYS. The challenge in using these method is properly setting up the problem. Small changes in boundary conditions and initial conditions can result in wildly inaccurate results.
